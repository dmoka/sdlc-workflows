Cline's most-shared community pattern: a curated set of five Markdown files under `memory-bank/` that the agent reads at the start of every session. Works in any agent, Cline just has the cleanest UX for it (auto-read on session start). The mechanic generalizes: any agent that respects custom instructions will follow the same rule.

## Starter templates

Create `memory-bank/` in your repo with these five files. Keep each short, the agent reads all five on every session start.

**`projectbrief.md`**, what this is

```markdown
[Project name]. [One-paragraph purpose.]
Primary stakeholders: [...]
Hard constraints, anything that must not change: [...]
```

**`productContext.md`**, why we're building it

```markdown
The user problem in their own words.
The shape of the solution.
What success looks like.
Explicit out-of-scope.
```

**`activeContext.md`**, what's happening right now

```markdown
Working on: [...]
Last completed PR: [...]
Decision pending: [...]
Files in mid-edit: [...]
Blocked on: [...]
```

**`systemPatterns.md`**, how the code is organized

```markdown
Architecture in one paragraph.
The 3-5 idioms this repo uses consistently.
Where common decisions tend to land.
```

**`techContext.md`**, the runtime stack

```markdown
Stack: [...]
Build / test / lint commands.
Local-dev setup.
CI shape.
External services this repo talks to.
```
