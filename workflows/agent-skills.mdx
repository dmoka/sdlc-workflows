Addy Osmani's take on agentic SDLC: 7 slash commands that map directly to the development lifecycle, each backed by skills the agent auto-activates based on what you're doing. The thesis: **encode the workflows, quality gates, and best practices that senior engineers use, so AI agents follow them consistently**.

40k+ stars (Feb 2026 release), MIT, and Addy's strong frontend/eng brand. Built for Claude Code primarily, with Gemini CLI and OpenCode support.

| What you're doing | Command | Key principle |
|---|---|---|
| Define what to build | `/spec` | Spec before code |
| Plan how to build it | `/plan` | Small, atomic tasks |
| Build incrementally | `/build` | One slice at a time |
| Prove it works | `/test` | Tests are proof |
| Review before merge | `/review` | Improve code health |
| Simplify the code | `/code-simplify` | Clarity over cleverness |
| Ship to production | `/ship` | Faster is safer |

Skills also activate automatically based on what you're doing, designing an API triggers `api-and-interface-design`, building UI triggers `frontend-ui-engineering`, and so on.

## Why it works

- **Skills, not just commands.** The 7 slash commands are the front door; behind them, specialized skills auto-activate based on the work you're doing (API design, UI engineering, debugging, etc.).
- **Production-grade defaults.** "Spec before code", "tests are proof", "clarity over cleverness", the principles are baked in, not optional.
- **Linear handoff.** Output of one phase is input to the next, `/spec` writes a PRD that `/plan` reads; `/plan` writes tasks that `/build` executes.

**Use for** production work where you want SDLC rigor without re-explaining it every session. Particularly strong for shipping features end-to-end with the discipline of a senior engineer.

**Avoid for** quick scripts where 7 phases is overkill. For exploratory spike work, see [grill-prototype-rewind](/workflows/grill-prototype-rewind) instead.
