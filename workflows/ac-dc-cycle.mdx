A four-stage cycle proposed by Tariq Shaukat, CEO of Sonar, in March 2026: **Guide → Generate → Verify → Solve**. The thesis is that traditional CI assumes "frequent, small developer commits," while agents instead "operate in asynchronous batches, often working for hours before dropping massive, complex payloads of code." The legacy CI factory model breaks under that load.

Each stage has a sharp definition:

- **Guide** — agents receive context (codebase shape, standards, requirements) before they generate
- **Generate** — AI writes code toward specified outcomes
- **Verify** — generated code is tested against quality and security standards
- **Solve** — issues trigger automated remediation; lessons feed back into the next cycle's Guide step

In Shaukat's framing, the developer's role shifts from primary author to **agent supervisor and quality governor** — defining the standards automated systems must meet rather than writing line-by-line.

**Use for** any organization scaffolding governance around a single-agent loop. AC/DC describes the gates between Generate and merge, not the inner loop itself — pair it with Ralph, PIV, or Compound Engineering for the actual code-writing step.

**Avoid for** solo prototyping where the verification + remediation overhead exceeds the work itself.
