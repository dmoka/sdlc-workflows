A composed-skills workflow from Matt Pocock that interleaves talking-through and trying-things. The pattern lives at the seam between two activities that usually fight each other: design conversation (Q&A surfacing unknowns) and prototyping (code that answers questions only experimentation can).

The flow is six beats:

1. **`/grill-with-docs`** — start a grilling session about a new piece of UI or API. The skill asks questions until misalignment is squeezed out of the design.
2. **Hit a question only code can answer** — "would this animation feel right?", "does the cache key collide in this case?", "what does the API look like for both modes?" Grilling can't resolve these by talking.
3. **`/prototype`** — branch into a throwaway code spike. Burn tokens. Iterate on the prototype freely until the question is answered.
4. **Iterate to confidence** — the prototype is a probe, not the deliverable. The point is the learning, not the code.
5. **`/rewind` + Summarize** — in Claude Code, rewind the conversation to the grilling turn where the question came up. Pick "Summarize" → "summarize what we learned from prototyping." The summary folds the new knowledge into the main thread.
6. **Continue grilling** with the new context. The prototype files remain on disk — they're a future reference, not a load on the conversation.

The two skills come from [mattpocock/skills](https://github.com/mattpocock/skills) (MIT, one-line install with `npx skills@latest add mattpocock/skills`). The `/rewind` mechanic is Claude Code-specific, but the *pattern* is portable: in any agent, you can manually summarize the prototype session and paste the summary back into the grilling thread before continuing.

**Use for** UI or API design where the right shape isn't obvious — you need to talk through tradeoffs *and* try things, not just one.

**Avoid for** mechanical changes with a known shape (refactors, renames, dep upgrades). Grilling overhead is pure tax when there's nothing to grill about.

The composition matters more than the individual pieces. `/grill-with-docs` alone is great. `/prototype` alone is great. Cycling between them — and using rewind to keep the conversation thread coherent — is what makes the loop pay off.
