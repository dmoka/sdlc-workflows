A vendor reference architecture published by Microsoft in February 2026 describing an end-to-end agentic SDLC stitched together from GitHub Copilot async agents on the front and Azure AI Foundry on the back. The shape is essentially **Orchestrator-Workers wearing Microsoft branding** — but for Azure-committed teams, the vendor support story is the actual differentiator.

The pipeline names four agents along the SDLC:

- **Planning Agent** — natural-language requirements turn into structured tickets
- **Coding Agent** — GitHub Copilot's async coding agent picks up a ticket, opens a PR
- **Quality Agent** — review, security scan, test coverage check
- **SRE / Deployment Agent** — proactive monitoring, automated rollback on regression

Human-in-the-loop gates sit between each pair. The glue is **GitHub Agentic Workflows** (natural-language workflow definitions on the GitHub side) and **Azure AI Foundry** (orchestration runtime for complex multi-step flows on the Azure side).

**Use for** Azure-committed enterprises that want vendor-blessed reference shapes rather than hand-rolling something. The named-agent pipeline is generic enough to translate to other stacks, but the value here is the integration story — Foundry, GitHub Actions, Copilot all wired together.

**Avoid for** small teams or non-Azure stacks. The orchestration cost only makes sense at the scale where you already have an Azure footprint and a platform team.
