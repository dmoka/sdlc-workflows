A deliberate two-layer split published by Daniel Sheard at McKinsey QuantumBlack in February 2026. The top layer — a **deterministic rule-based orchestrator** — has no LLM decision-making at all on phase transitions. The bottom layer — **bounded specialist agents** — only acts inside whichever phase the orchestrator is currently in.

Five named agents sit in the bottom layer:

- **Requirements** — break down features into testable units
- **Architecture** — design and tradeoff choices
- **Coding** — implementation
- **Critic** — validation, judgment calls, escalation
- **Knowledge** — context queries, assumption tracking

The on-disk shape is the spec:

```
.sdlc/
├── context/      # codebase + standards loaded into the Guide step
├── specs/        # requirements + architecture + tasks
└── knowledge/    # tracked assumptions, decisions, open questions
```

Reusable instructions live as **`SKILL.md` files** that encode domain expertise as machine-readable workflow specs. The folder structure itself is the spec.

**Use for** production work in regulated or high-stakes settings where phase transitions must be predictable and auditable. QuantumBlack sits at the **opposite pole from Ralph** on the autonomy axis — Ralph trusts the model with everything; QuantumBlack only trusts it inside tight phase walls.

**Avoid for** exploratory work or small teams. The orchestrator scaffolding pays off only when audit trails and bounded agent behavior are hard requirements.
