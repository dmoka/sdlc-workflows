A Markdown file at the repo root (uppercase filename — `AGENTS.md`) that agents read before doing anything else. Co-released in 2025 by Google, OpenAI, Factory, Sourcegraph, and Cursor as a vendor-neutral standard, now stewarded by the Linux Foundation's Agentic AI Foundation.

The standard sections most teams ship are short:

- **Build & Test** — the exact commands to run; no narrative
- **Architecture Overview** — one paragraph; one diagram if needed
- **Security** — what's sensitive; what the agent must not touch
- **Git Workflows** — branch naming, commit format, PR rules
- **Conventions & Patterns** — the small set of in-repo idioms

Critically, agents resolve AGENTS.md **nearest-wins**: per-package overrides at `packages/foo/AGENTS.md` take precedence over the repo-root one for work inside `packages/foo`. That's the part most adopters miss — it lets large monorepos guide agents at every level instead of forcing one giant root file.

**Use for** any repo more than one agent will touch. The cost is one file; the payoff is every agent in every tool starts from the same context.

**Avoid for** single-file scripts where one inline system prompt covers everything. AGENTS.md is overkill below a certain repo size.
