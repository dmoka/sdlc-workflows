AWS Labs' formal methodology for AI-driven development. **Three-phase adaptive workflow**, inception, construction, operations, with steering rules that adapt to the work in front of the agent, quality gates between phases, and explicit human oversight checkpoints. MIT-0 licensed (one of the most permissive open-source licenses), backed by an [AWS DevOps blog post](https://aws.amazon.com/blogs/devops/ai-driven-development-life-cycle/) and a method-definition paper.

The thesis: agentic SDLC needs **adaptive steering** (rules that change with the phase you're in) rather than a fixed prompt or a single set of skills. The same agent behaves differently in inception (more questions, more spec-writing) vs. construction (more code, more verification) vs. operations (more monitoring, more remediation).

## Three phases

- **Inception**, requirements + design. The agent leads with questions, spec writing, and architecture decisions. Quality gate: spec approved by a human before construction begins.
- **Construction**, implementation + verification. The agent writes code, runs tests, runs static analysis (Bandit, Checkov, Gitleaks, Grype, Semgrep, all configured in the repo). Quality gate: tests + security scans clean before operations.
- **Operations**, deploy + monitor + remediate. The agent handles ongoing operations: deploy through Q Developer/Kiro/your CI, watch metrics, and auto-remediate when steering rules say it's safe.

Each phase has its own rule set in `aidlc-rules/`, conditionally referenced based on what the agent is doing.

## Multi-tool support

The rule pack ships per-platform setup for: **Kiro · Amazon Q Developer IDE · Cursor · Cline · Claude Code · GitHub Copilot · OpenAI Codex**, plus a generic "Other Agents" path. Same methodology across harnesses.

**Use for** production-grade SDLC where you want a documented methodology, AWS-vetted quality gates, and explicit phase transitions. Particularly relevant for enterprise teams that already use AWS tooling (Q Developer, Bedrock, Amplify).

**Avoid for** hobby projects or one-off scripts. Three-phase ceremony + the rule pack is overhead unless you have real production constraints.
